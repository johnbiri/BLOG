Diário de Estudos - #002

Aprendendo a usar Ruby on Rails para poder participar de projetos:

Usando uma máquina virtual no Cloud9 (URL: http://localhost:8181/ide.html)

1. Acessar o terminal no Git Bash
2. Ir para o projeto com o comando: cd /c/Projetos/NOME_DO_PROJETO
3. Dentro da pasta digite o comando: vagrant up

Notas:
- vagrant up -> Liga a máquina
- vagrant suspend -> Pausa a máquina
- vagrant halt -> Desliga a máquina
- rvm use x.x.x --default -> Seta a versão como padrão
- rvm use ruby-x.x.x -> Usa a versão escolhida

A seguir comandos do Ruby Version Manager (RVM):
1. rvm list known -> Este comando lista todas as versões disponíveis no repositório
2. rvm get head -> Atualiza a lista
3. rvm list -> Lista as versões instaladas localmente
4. rvm install x.x.x -> Instala uma versão escolhida
5. rvm install x.x.x --default -> Instala uma versão escolhida e a torna padrão

Ruby, IRB e Pry:

1. Como executar um arquivo Ruby? -> ruby meu_arquivo.rb
2. Exemplo -> puts "Olá mundo"

Interactive Ruby Shell (IRB) é uma ferramenta (REPL) de linha de comando que permite o desenvolvimento e execução de código Ruby de forma interativa, facilitando o aprendizado, a depuração e a prática do Ruby em tempo real.

No prompt digite irb Dentro do IRB você pode digitar qualquer instrução Ruby para ser interpretada. Tente:
- i1 -> 1+2
- i1 -> "curso da ruby on rails".reverse
- i1 -> a = {curso: "rails", versao: "5.x"}

Conhecendo o Pry (https://github.com/pry) - O Pry é uma alternativa ao IRB padrão e vem com algumas melhorias como o syntax highlighting que destaca os elementos dos códigos como cores e a auto-indentação.

Instalação do Pry -> gem install pry

Tornando Pry padrão ao chamar o IRB:

1. Crie o arquivo .irbrc na home do usuário: touch ~/.irbrc
2. Após criar o arquivo edite ele no terminal com o Vim: vim ~/.irbrc
ou com o Nano: nano ~/.irbrc

- require 'rubygems'
- require 'pry'
- Pry.start
- exit

15 de dezembro de 2024


Eu criei e publiquei um site estático no GitHub Pages para aplicar meus conhecimentos em HTML, CSS, Bootstrap e GitHub. Primeiro, desenvolvi um site básico com uma página principal (index.html) usando meu editor de código preferido, o Visual Studio Code (VSCode). Após finalizar o site, criei um novo repositório no GitHub e o nomeei de acordo com o propósito do meu site, por exemplo, “BLOG”. Em seguida, adicionei o repositório remoto ao meu projeto com o comando git remote add origin [URL do meu repositório] e fiz o upload dos arquivos com git push origin main. Verifiquei se os arquivos foram corretamente enviados para o GitHub e ativei o GitHub Pages em Settings > Pages, selecionando a branch “main” como a fonte e clicando em “Save”. Após alguns minutos, copiei o URL fornecido pelo GitHub e testei o site em meu navegador para garantir que tudo estava funcionando como esperado. Por fim, compartilhei o URL do meu site com meus colegas de classe. Esta atividade me proporcionou uma prática valiosa na criação e publicação de sites, solidificando meus conhecimentos em desenvolvimento web e me familiarizando com ferramentas essenciais do desenvolvimento moderno.
